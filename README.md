## What gets deployed

| Symlink | What it configures |
|---|---|
| `~/.config/nvim` | Neovim config (LazyVim-based) |

That's it. Just one symlink. But the config inside `nvim/` is full LazyVim with custom plugins, LSP setup, and personal keybinds.

## Install

### As part of the umbrella (recommended)

```bash
git clone --recurse-submodules https://github.com/blak0p/dotfiles.git ~/dotfiles
cd ~/dotfiles
./install.sh --nvim
```

### Standalone (without the umbrella)

```bash
git clone https://github.com/blak0p/dotfiles-editors.git
cd dotfiles-editors
./install.sh
```

### Install system dependencies first

The `install.sh` only creates the symlink тАФ it does NOT install `neovim` or its tool dependencies. Use the bundled `install-deps.sh`:

```bash
bash deps/install-deps.sh
```

It auto-detects your package manager (`pacman`, `dnf`, or `apt`) and installs `neovim`, `lazygit`, `nodejs`, `ripgrep`, `fd`, `gcc`, `make`, etc. See [Dependencies](#dependencies).

## Update

```bash
cd ~/dotfiles/dotfiles-editors
git pull
./install.sh   # idempotent
```

Or as part of the umbrella:

```bash
cd ~/dotfiles
git submodule update --remote --merge dotfiles-editors
```

After updating, open nvim and run `:Lazy sync` to update plugins.

## Uninstall / rollback

```bash
rm -f ~/.config/nvim

# Restore from latest backup (if any)
BACKUP=$(ls -td ~/.dotfiles-backup-* | head -1)
[ -n "$BACKUP" ] && [ -d "$BACKUP/.config/nvim" ] && cp -a "$BACKUP/.config/nvim" ~/.config/
```

## Customize

The most common edits:

- **Add a plugin**: edit `nvim/lua/plugins/<name>.lua` (LazyVim convention) or add to `lazyvim.json` extras
- **Change keybinds**: edit `nvim/lua/config/keymaps.lua` or your custom layer
- **Change colorscheme**: edit `nvim/lua/plugins/colorscheme.lua`
- **Add an LSP**: LazyVim's `:LspInstall` or edit `nvim/lua/lsp/<server>.lua`
- **Add a formatter**: LazyVim's `:Format` will pick up null-ls / conform.nvim configs

Inside nvim, `:Lazy` opens the plugin manager UI. `:Mason` for LSPs. `:checkhealth` for diagnostics.

## Troubleshooting

### `nvim` not found

Install it via the system package manager:

```bash
# Arch
sudo pacman -S neovim
# Fedora
sudo dnf install neovim
# Debian 13+ / Ubuntu 25.04+
sudo apt install neovim
```

Older Ubuntu versions need a PPA. See `neovim.io`.

### Plugins won't sync

```vim
:Lazy clean
:Lazy update
```

### LSP not attaching

```vim
:LspInfo          " check what LSP is loaded for current buffer
:Mason            " install missing LSPs
:checkhealth      " run diagnostics
```

### Spell files getting committed

The repo's `.gitignore` already excludes `nvim/spell/*.spl` and `nvim/spell/*_words.txt` (regenerated by nvim). If you see them in `git status`, the gitignore isn't being respected тАФ re-check it with `git check-ignore -v nvim/spell/en.utf-8.spl`.

### `fd-find` / `fdfind` issues on Debian

Debian installs the `fd-find` package with the binary named `fdfind` (with symlink to `fd`). LazyVim's `telescope-fzf-native` handles both. If not:

```bash
# Force the symlink
sudo ln -sf $(which fdfind) /usr/local/bin/fd
```

## Dependencies

- Arch: `deps/packages.txt`
- Fedora: `deps/packages.dnf.txt`
- Debian/Ubuntu: `deps/packages.apt.txt`

Or `bash deps/install-deps.sh` (auto-detects).

Notable packages:
- `neovim` тАФ the editor
- `git`, `curl` тАФ for plugin management
- `ripgrep`, `fd-find` тАФ for Telescope fuzzy finding
- `nodejs`, `npm` тАФ for LSPs that need Node (tsserver, eslint, etc)
- `gcc`, `make`, `unzip` тАФ to compile treesitter parsers and Mason tools
- `lazygit` тАФ TUI git client (used by LazyVim's git integration)
